**WishList**
- сайт для створення списків бажань та інтерактивного підбору подарунків.

**Призначення:** допомога людям систематизувати свої бажання та полегшити процес вибору подарунків для друзів і близьких.

**Основний функціонал:** 

-авторизація (особисті кабінети кожного користувача);

-менеджмент бажань (додавання/редагування/видалення подарунків зі свого списку);

-доступ до бажанок (генерація посилання, що надає доступ до перегляду);

**Технологічний стек:**

*Мова:* Python

*Фреймворк:* Django

*База даних:* SQLite (для розробки)

*Стилі:* CSS (Bootstrap 5) для адаптивності.

**Шлях користувача.**

Новий користувач потрапляє на головну сторінку. де бачить тематичну ілюстрацію та текст(до прикладу: "Створюйте, діліться та отримуйте подарунки, які Вам подобаються!"), а також інтерфейс із категоріями:  Wishlist - логотип (перехід на початкову сторінку).

  *Features* - опис сайту.
        
  *Examples* - публічні WishList (розділ з відкритим доступом до бажанок - або прописані списки, або списки реальних користувачів, що надали дозвіл на публічність своїх бажань).
              
  *About* - про нас (основна інформація про засновників сайту)*
              
*ця категорія не є обов'язковою та не несе ніякого практичного застосування для користувачів.

  *Get Started/ Create Your First WishList* - перехід до реєстрації та створення листа бажань.
              
  *Ілюстрація* - фото, яке буде підходити під тематику сайту.
              
  *Основний текст* - фраза, що буде підходяща для теми подарунків та бажань.

Для реалізації поділимо роботу на етапи:

  **1 етап.** Ініціалізація
  
  **2 етап.** Створення моделей (описати таблицю Gift; зв'язати її з вбудованим користувачем).
  
  **3 етап.** Перевірка (зареєструвати моделі, щоб перевірити додавання подарунків).
  
  **4 етап.** Налаштування шаблонів (base.html, де буде меню категорій, підключити Bootstrap 5).
  
  **5 етап.** Реєстрація та вхід(використати вбудовані функції, створити просту сторінку реєстрації).
  
  **6 етап.** Оформити головну сторінку.
  
  **7 етап.** Мій список (вивожить лише подарунки поточного користувача).
  
  **8 етап.** Додавання подарунка (створити сторінку з формою, де користувач вводить дані).
  
  **9 етап.** редагування та видалення (додати функцію зміни або видалення запису).
  
  **10 етап.** Публічний доступ (створити окремий URL, де будь-хто може побачити список, але приховати кнопки редагування).
  
  **11 етап.** Реалізація додаткових функцій.

  *Додаткові функції.*
    
  Тест, що допомагає вибрати подарунок. 
    
  **Ідея:** допомагає структурувати думки, пропонуючи варіанти на основі конкретних параметрів, таких як бюджет, хобі чи привід свята.
    
  **Технічний принцип:** Створюється невеликий каталог ідей, і програма просто відсікає зайве. Наприклад, якщо користувач обрав "Спорт", система приховує ідеї з категорій "Книги" чи "Кулінарія". Користувач проходить тест, відповідаючи на прості питання. Замість того, щоб просто шукати збіги в базі, програма формує промпт для ШІ.
    
  **1 етап:** Користувач обирає базові параметри (Бюджет, Категорія, Привід). Система відсікає невідповідні варіанти з наперед заготовленої бази ідей.
    
  **2 етап:** Програма бере відповіді користувача і формує динамічний промпт.
    
  **3 етап:** ШІ повертає варіанти, які відображаються у вигляді карток.
    
  **Технічна реалізація:**

Створити форму (Питання з варіантами відповідей).

Інтегрувати API: Використати бібліотеку (наприклад, openai або google-generativeai).

Зв'язати з БД: Додати кнопку "Додати до вішліста", яка бере текст, згенерований ШІ, і записує його в таблицю Gift.
